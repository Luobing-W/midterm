
# 🗄️ Database Schema & Codebook

## Overview
This document describes the database structure for the Park Injury Reporting System. The application uses a **Supabase PostgreSQL** database with one main table.

---

## 📋 Table: `park_injuries`

**Purpose:** Stores all reported park injury incidents submitted through the Shiny application.

### Table Structure

| Column Name | Data Type | Constraints | Description | Example Value |
|-------------|-----------|-------------|-------------|---------------|
| `id` | BIGINT | PRIMARY KEY, AUTO INCREMENT | Unique record identifier | `1`, `2`, `3` |
| `report_date` | DATE | NOT NULL | Date when the incident occurred | `2024-01-15` |
| `park_name` | TEXT | NOT NULL | Name of the park where incident happened | `Central Park` |
| `location_detail` | TEXT | NOT NULL | Specific location within the park | `Playground`, `North Trail` |
| `injury_type` | TEXT | NOT NULL | Category of injury sustained | `Fall`, `Sprain`, `Collision` |
| `severity_level` | INTEGER | CHECK (1-5) | Severity rating scale | `1` (minor) to `5` (critical) |
| `description` | TEXT | NULLABLE | Detailed text description of incident | `Child fell off swing` |
| `created_at` | TIMESTAMPTZ | DEFAULT NOW() | Timestamp when record was created in database | `2024-01-15 14:30:00 UTC` |

---

## 📊 Variable Codebook

### `id`
- **Type:** BIGINT (64-bit integer)
- **Range:** 1, 2, 3, ... (auto-incrementing)
- **Purpose:** Primary key for unique identification
- **Generated:** Automatically by database

### `report_date`
- **Type:** DATE
- **Format:** YYYY-MM-DD
- **Range:** Any valid date
- **Purpose:** When the injury incident occurred
- **Input:** User selects via Shiny `dateInput`

### `park_name`
- **Type:** TEXT (string)
- **Allowed Values:** 
  - `Central Park`
  - `Riverside Park`
  - `Sunset Park`
  - `Liberty Park`
- **Purpose:** Identify which park the incident occurred at
- **Input:** User selects via Shiny `selectInput`

### `location_detail`
- **Type:** TEXT (string)
- **Max Length:** Unlimited (PostgreSQL TEXT)
- **Purpose:** Specific area within the park
- **Examples:** `Playground`, `Basketball Court`, `Hiking Trail`, `Tennis Court`
- **Input:** User types via Shiny `textInput`

### `injury_type`
- **Type:** TEXT (string)
- **Allowed Values:**
  - `Fall` - Slipping, tripping, falling from height
  - `Sprain` - Twisted ankle, wrist, etc.
  - `Collision` - Hit by object or person
  - `Cut` - Laceration from sharp object
  - `Other` - Any other injury type
- **Purpose:** Categorize the type of injury for analysis
- **Input:** User selects via Shiny `selectInput`

### `severity_level`
- **Type:** INTEGER
- **Range:** 1 to 5 (enforced by CHECK constraint)
- **Scale:**
  | Value | Meaning | Description |
  |-------|---------|-------------|
  | 1 | Minor | No medical attention needed |
  | 2 | Light | Basic first aid required |
  | 3 | Moderate | Medical attention recommended |
  | 4 | Serious | Emergency care needed |
  | 5 | Critical | Hospitalization required |
- **Purpose:** Quantify injury severity for prioritization
- **Input:** User selects via Shiny `numericInput`

### `description`
- **Type:** TEXT (string)
- **Max Length:** Unlimited
- **Purpose:** Free-text details about the incident
- **Examples:** 
  - `Child fell off swing and landed on grass`
  - `Ankle twisted on uneven rock on trail`
  - `Two players collided during basketball game`
- **Input:** User types via Shiny `textInput`

### `created_at`
- **Type:** TIMESTAMPTZ (Timestamp with Time Zone)
- **Format:** ISO 8601 with timezone
- **Purpose:** Track when the report was submitted to the system
- **Generated:** Automatically by database (DEFAULT NOW())

---

## 🔧 SQL Table Definition

```sql
CREATE TABLE park_injuries (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    report_date DATE NOT NULL,
    park_name TEXT NOT NULL,
    location_detail TEXT NOT NULL,
    injury_type TEXT NOT NULL,
    severity_level INTEGER CHECK (severity_level BETWEEN 1 AND 5),
    description TEXT,
    created_at TIMESTAMPTZ DEFAULT NOW()
);

-- Indexes for query optimization
CREATE INDEX idx_park_name ON park_injuries(park_name);
CREATE INDEX idx_report_date ON park_injuries(report_date);
